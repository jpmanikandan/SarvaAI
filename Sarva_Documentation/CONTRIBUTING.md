# Contributor Guide

Thank you for your interest in contributing to Sarva AI!

## 1. Getting Started
1.  **Fork** the repository.
2.  **Clone** your fork: `git clone https://github.com/your-username/sarva-ai.git`.
3.  **Branch**: Create a feature branch (`git checkout -b feature/amazing-feature`).

## 2. Coding Standards
- **Java**: Follow standard Java naming conventions (CamelCase for classes, camelCase for methods).
- **Comments**: JavaDoc is required for public methods.
- **Formatting**: Use the Eclipse formatter profile provided in `eclipse-formatter.xml`.

## 3. Pull Request Process
1.  **Update Documentation**: If you change functionality, update the relevant docs (`API_Reference.md` etc.).
2.  **Pass Tests**: Ensure `mvn test` passes locally.
3.  **Submit PR**: Open a Pull Request against the `main` branch. Provide a clear description of your changes.

## 4. Reporting Bugs
Use the GitHub Issues tab to report bugs. Include:
-   Steps to reproduce.
-   Expected vs Actual behavior.
-   Logs/Screenshots.
